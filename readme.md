# Brno Public Transport Departure Board

A minimalistic mobile-friendly departure board for public transport in Brno, Czechia. Enter a stop name to see upcoming departures, filter by specific routes, and bookmark your frequent stops.

## Project Structure
```txt
.
├── data/
│   ├── GTFS/            # Unzipped GTFS .txt files
│   ├── stops.csv        # Brno stops dataset (Actually, there are stops from entire South Moravian Region)
│   └── transit.db       # SQLite database (generated by ingest.py)
├── src/
│   ├── static/          # Frontend assets (HTML, CSS, JS)
│   ├── ingest.py        # GTFS to SQLite ingestion script
│   ├── app.py           # Flask Backend API (also handles Frontend)
│   ├── start.sh         # Container startup script
│   └── requirements.txt # Python dependencies
├── Dockerfile           # Docker build instructions
└── docker-compose.yml   # Docker Compose configuration
```

## Setup & Running

### Option 1: Running via Docker Compose (Recommended)
This handles environment variables and volumes automatically.
```bash
docker-compose up --build
```
Access at `http://localhost:5000`.

---

### Option 2: Running Locally
**Prerequisites:**
- Python 3.12.3+
- `pipenv`

**Windows Warning:**
This project uses the `zoneinfo` module for strict Czechia time handling. On **Windows**, you must install `tzdata`:
```bash
pipenv install tzdata
```

**Steps:**
1. Install dependencies:
   ```bash
   pipenv install
   ```
2. Ingest data (ensure `data/GTFS` and `data/stops.csv` exist):
   ```bash
   cd src
   pipenv run python ingest.py
   ```
3. Run Server:
   ```bash
   pipenv run python app.py
   ```
Access at `http://localhost:5000`.

---

### Option 3: Running via Docker Standalone
1. Build the image:
   ```bash
   docker build -t brno-transit .
   ```
2. Run the container (mounting the data volume is required for ingestion/persistence):
   ```bash
   docker run -p 5000:5000 -v $(pwd)/data:/app/data brno-transit
   ```
